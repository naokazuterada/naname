# 先生をプログラミングしてお菓子をゲットしよう

授業で使ったプレゼン資料です。

「チャレンジステージ2」は時間の関係で省略しました。本当は、このステージまで行けるとプログラミングの凄さがわかると思ったのですが、どうしても時間が足りませんでした。

### 授業のレシピ

今回は25人の参加者がいたので、1チーム5人の5チームに分かれてもらいました。チームの人数は、多すぎると参加しにくい子が出てくるかもしれないので、これくらいがマックスかなと思います。
各チームには、一人ずつアシスタントの先生をつけてロボット役になってもらいます。

#### ステージの準備

チーム毎に教室の床にマスキングテープで5×5のマス目、その横に大きな枠1つ、小さな枠1つを用意します。
![図]()

#### 道具の準備

##### 正方形のダンボール板
1つは片面に矢印を描いて「スタート位置」として使います。その他は「ブロック」として立ち入りできないエリアとして使います。
マス目にそって置けるように、マス目の大きさに合わせたいのですが、逆にダンボールの大きさに合わせてマス目の大きさを決めるといいと思います。
（写真）
として
今回は、[このA3サイズの板ダンボール](https://www.amazon.co.jp/gp/product/B01LY375EH/)を使って、297x297mmの正方形にしました。


##### 色付きの棒（2色）
お菓子とジュースを表すために2色用意してください。
棒でなくても、紙コップに入れば色紙を切ったものなどで構いません。
本番では5本1セットでやったのですが、各10本ずつあればOKです。
![写真]()

##### 紙コップ、紙皿
1チーム10個ずつ程度。
紙コップに、さきほどの棒を入れてステージに配置します。
紙皿は、紙コップの上にかぶせて蓋にできればよいので、画用紙などで代用可能です。かぶせた時に上になる面に「？」と描いておきましょう。
![写真]()

##### 命令カード
このカードを使って、プログラミングします。下記の数を用意しましたが、最終ステージでは足りなくなって、予備として作っていたものを配ったのでもう少し多めの方が良さそうです。
![写真]()

| 命令                    | 枚数        | 色       |
|:-----------------------|:-----------|:---------|
| たべる                  | 10         | 黄色 |
| のむ                    | 10        | オレンジ |
| すすむ                  | 24         | 緑 |
| 右むけ右                 |  24       | 黄緑 |
| くりかえし(  ) {         | 3          | ピンク |
| }                      |            | ピンク |
| もし(   ) {             | 3          | 青 |
| } それ以外で、もし(  ) {   |            | 青 |
| } それ以外なら {          |            | 青 |

| カスタム {               | 3           | グレー |
| }                      |             | グレー |
| カスタム（実行用）         | 6           | グレー |

2枚以上でセットになるカードは少し長めにしておくと、並べた時に分かりやすいです。

シールを貼るのが大変な場合は、シートとペンの代わりに鉛筆と消しゴムでも良いです。

### 遊び方

そのステージを別に用意しておいたカードで大きな枠内にプログラミングし、ロボットはその通りに動きながらステージ上のアイテムをゲットしていきます。

---

プログラムは「表現」なのか？

プログラミングというのは機械に対する命令です。人間はプログラムを通して望む動作を機械に伝え、機械は与えられた通りの動作を行います。
